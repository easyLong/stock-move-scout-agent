# 竞价详情系统

本文记录集合竞价独立页面的目标、数据链路、字段定义和页面展示规则。竞价详情不再混入异动情报流，只作为 `/auction-detail` 的专门研究系统。

## 目标

竞价阶段最重要的是快速识别两件事：

- 机会：涨停封单最强 Top3 个股，判断竞价机会集中在哪。
- 风险：跌停封单最强 Top3 个股，判断竞价风险集中在哪。

同时保留 09:15-09:25 的封单过程，观察动态博弈：

- 09:19-09:20：最后可撤单窗口，重点看撤单或继续加强。
- 09:20-09:25：不可撤单后，重点看封单继续加强还是被卖压消耗。

## 数据链路

```text
09:15-09:25 集合竞价封单雷达
  -> auction_minute_analysis
  -> auction_candidates
  -> auction_trend_summary
  -> /auction-detail
```

| 表 | 职责 |
| --- | --- |
| `auction_minute_analysis` | 分钟封单雷达，采集所有涨停封单和跌停封单 |
| `auction_candidates` | 最终涨停封单候选，只保留 Top3 |
| `auction_trend_summary` | 把分钟雷达压缩成页面可读摘要 |

历史数据只能基于当时已经保存的分钟明细补 `auction_trend_summary`，不能倒采当时没有保存的竞价快照。

## 字段口径

| 字段 | 含义 |
| --- | --- |
| 当前封单 | 当前最新分钟的涨停侧或跌停侧一档封单金额 |
| 19封单 | 09:19 附近封单金额，用来观察可撤单窗口前状态 |
| 20封单 | 09:20 附近封单金额，用来观察不可撤单开始时状态 |
| 19-20 | `20封单 - 19封单`，看最后可撤单窗口是加强还是撤单 |
| 20-最新 | `当前封单 - 20封单`，看不可撤单后是继续加强还是消耗 |
| 当前涨幅 | 最新分钟竞价涨跌幅，按 A 股涨红跌绿显示 |
| 当前成交额 | 09:25 集合竞价成交额，只作为辅助确认字段 |
| 状态 | 根据 19-20 和 20-最新 组合生成的过程标签 |

页面判断主轴是封单额变化，不再使用“竞价额变化”作为核心指标。

## 页面展示规则

页面主列表只显示 6 只票：

- 机会 Top3：涨停封单方向。
- 风险 Top3：跌停封单方向。

榜单依据可以切换：

- 当前封单
- 19封单
- 20封单
- 19-20
- 20-最新

切换依据后，Top3 重新排序。默认看当前封单。

## 颜色规则

机会和风险的颜色语义不同：

| 方向 | 加强 | 走弱 |
| --- | --- | --- |
| 涨停机会 | 淡红 | 淡绿 |
| 跌停风险 | 淡绿 | 淡红 |

`19-20`、`20-最新`、`状态` 都按这个方向语义着色。`当前涨幅` 按 A 股习惯独立显示：正涨幅淡红，负涨幅淡绿。

## 手工补数

```powershell
python scripts\build_auction_trend_summary.py --trade-date YYYY-MM-DD --limit 80 --mysql-enabled --mysql-password <MYSQL_PASSWORD>
```

正常交易日会在 09:26 自动运行 `auction_trend_summary`。手工命令主要用于历史补数和排查。
