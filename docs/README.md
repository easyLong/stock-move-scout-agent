## 一句话架构

项目围绕“个股异动，快速定位异动原因，找真正有效证据”组织：盘后生成研究池和证据底稿，盘中只做热数据扫描和动态展示，页面优先读取缓存和快照，避免临场拼复杂证据。

```text
研究池
  -> 盘后确认数据
  -> 有效事实与模型总结
  -> 领头羊快照 / 根证据缓存
  -> 盘中异动扫描 / 市场概览
  -> Web 展示
```

## 维护原则

- 文档只描述当前主流程，废弃链路单独标注。
- 数据源先落原始事实，再进入可读层、缓存层和展示层。
- 盘中热数据和盘后冷数据分开，避免页面实时计算过重。
- 模型只处理已经进入有效事实层的候选事实。
- 没有有效事实时不调用模型。
