## 当前一句话架构

项目的本质是：围绕研究池股票，在盘中快速发现异动，并把异动和真正还有效的事实、题材角色、市场背景连接起来，尽量让证据详情一眼能看出“为什么动、证据硬不硬、是否有延续性”。

核心链路：

```text
研究池
  -> 行情扫描 / 市场概览 / 题材关系
  -> 异动事件与判断
  -> F10 近期重要事件
  -> 有效事实
  -> 模型总结
  -> 根证据缓存
  -> Web 展示
```

## 维护原则

- 主流程只写当前真实使用的链路，废弃内容单独标注。
- 数据源先落原始事实，再进入可读缓存或展示层。
- 盘中热数据和盘后/盘前冷数据分开，避免页面实时计算过重。
- UI 优先读取缓存；模型总结只在有效事实存在且内容变化时运行。
